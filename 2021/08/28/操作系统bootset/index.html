引导扇区的作用：启动设备的第一个扇区，启动设备信息被设置在CMOS中。硬盘的第一个扇区存储着开机后执行的第一段可以控制的代码：

1、第一段代码

.text等是伪操作，是告诉编译器产生文本段，用于标识文本段的开始位置。

  • .text：表示以下是代码段
  • .data：表示以下是初始化数据段
  • .bss：表示以下是未初始化数据段

详见https://blog.csdn.net/qqliyunpeng/article/details/45116615

begtext相当于跳转，这也就意味着从.text跳到.data，再跳到.bss，也就是说.text、.data、.bss这三个段是重叠的，并没有分段。。

ax、bx、cx、dx四个寄存器是通用寄存器，用于存放数据。

分析这段代码：

  • ```assembly
    mov ax,#BOOTSEG
    mov ds,ax
    mov ax.#INITSEG
    mov es,ax

    sub指令是相减，所以结果就是si = 0，di = 0。

    ds与si构成地址：07c0:0000（7c00）；es与di构成地址：9000:0000（90000）

  • ```assembly
    rep movw

    jmpi是间接跳转指令。go是一个标号，代表着ip，指偏移，表示从程序开始一直到该段的地址长度。执行的结果最后是：cs = INITSEG，ip = go

2、第二段代码

0x13是BIOS读磁盘区的中断，也就是说出现了0x13，就会按照下面的规则将指定扇区的代码加载到内存的指定位置，它的参数有：

  • AH：02
  • AL：读取扇区数
  • CH：读取柱面数[0,79]
  • CL：读取扇区数[1,18]
  • DH：磁头数[0,1]
  • DL：驱动器（0x0～0x7f表示软盘，0x80～0xff表示硬盘）

所以代码中的意思就是：

  • ah = 0x0200是读磁盘，al = 扇区数量（SETUPLEN = 4），因为ax是16位的，它有高八位的ah和低八位的al，所以说
  • cx的高八位ch是柱面号，低八位是开始扇区，这里是存放的2，因为第一个扇区是引导扇区，所以从第二个扇区开始读
  • dx的高八位是存放的磁头号，低八位存放的是驱动器号
  • es与bx组成内存地址

所以说这段代码就是在读磁盘，读到哪儿？es:bx处，也就是90200处。

截至目前为止，我们知道了boot有一个扇区，setup有四个扇区

3、第三段代码

0x10中断是BIOS的显示服务，它接受的参数有：

  • AH：0x13
  • AL：显示模式，有四种：
    • 0x00：字符串只包含字符码，显示之后不更新光标位置，此时属性值在BL中
    • 0x01：字符串只包含字符码，显示之后更新光标位置，此时属性值在BL中
    • 0x02：字符串包含字符码及属性值，显示之后不更新光标位置
    • 0x03：字符串包含字符码及属性值，显示之后更新光标位置
  • BH：视频页
  • BL：属性值（前提是AL = 0x00或0x01）
  • CX：字符串长度
  • DH、DL：屏幕上显示起始位置的行列值
  • ES:BP：字符串的物理地址

再回过头来看上面的代码：

  • 先是13号中断的内容：获得驱动器号为0，磁盘参数为8

  • 接着是第一个10号中断，旨在读光标

    • ```assembly
      mov ah,#0x03

      要显示的字符长度是24个字符

    • ```assembly
      mov bx,#0x0007

      将偏移地址指向msg1处，bp寄存器是一个指针寄存器

    • ```assembly
      mov ax,#1301

  • 第一个.byte的内容是：回车、换行的ASCII码——回车是10，换行（CR）是13

  • ”Loading system …“共占18个by字符

  • 第二个.byte的内容是：连续两个回车换行

加起来一共24个字符，小于等于cx中的值，所以能够全部输出。
