CSAPP--Data Lab

前言

Data Lab是CMU在CSAPP这本书上的第一个实验，其中的所有编程都是基于的位运算，涉及的知识点包括位运算的使用、补码、反码、二进制等。

其下载地址为http://csapp.cs.cmu.edu/3e/datalab-handout.tar

在每次进行编程后都需要先使用make btest命令，在使用./btest去进行最后的检测评分。

2、tmin

题目解释：return minimum two's complement integer

返回最小的补数那就是0x80000000。

解答：最小的补数是0x80000000。因为正数的补码是其本身，负数的补码是该数的反码加1。最小的补码就自然而然地想到符号位为1，其余位全为0，十六进制下就是最高位为8，其余位为0

3、isTmax

题目解释：returns 1 if x is the maximum, two's complement number, and 0 otherwise

通过位运算计算x是否是补码最大值

解答：做这个题目就得先知道补码的最大值是多少。根据第二题，就知道最大值是除了符号位是0，其余位全是1，十六进制下也就是0x7fffffff。也就是要去判断x是不是这个。最简单的办法就是直接与这个最大值异或

4、allOddBits

题目解释：return 1 if all odd-numbered bits in word set to 1 where bits are numbered from 0 (least significant) to 31 (most significant)

判断所有奇数位是否都为1，这里的奇数指的是位的阶级是2的几次幂。重在思考转换规律，如何转换为对应的布尔值。例如0xAAAAAAAA就符合题目要求。

解答：这道题采用掩码的方式——利用位移运算符构造奇数位全是1的掩码，然后获取输入的x的奇数位，将其他位清零，最后与掩码进行异或，最后返回其值的逻辑非

5、negate

题目解释：return -x

返回输入值的相反数

解答：感觉没啥好说的

6、isAsciiDigit

题目解释：return 1 if 0x30 <= x <= 0x39 (ASCII codes for characters '0' to '9')

计算输入值是否是数字 0-9 的 ASCII 值。

解答：这是一道充分向你展示位运算强大的题。0 ~ 9的ASCII值在0x30 ~ 0x39之间，对应的十进制是48 ~ 57，需要使用两个标志位——UpperBound，加上比0x39大的数后符号由正变负；LowerBound,加上比0x30小的值时是负数。简单来说就是与0和9做差，根据差值的符号位来判断。

7、conditional

题目解释：same as x ? y : z

使用位级运算实现C语言中的 x?y:z三目运算符

解答：x为1则取y，否则取0。所以将x转换为全1或全0，再(x&y)或者(~x&z)时，一定有一个成立。返回的就是y或者z的值

8、isLessOrEqual

题目解释：if x <= y then return 1, else return 0

使用位运算来判断两个数的大小

解答：想一想不使用位运算怎么比较大小——符号不同正数大，符号相同两者做差，看差值的符号。位运算下也是基于的此规则。唯一的区别就在于位运算下是没有减法的，减法只能通过与补码做加法实现。

9、logicalNeg

题目解释：implement the ! operator, using all of the legal operators except !

使用位级运算求逻辑非 !

解答：逻辑非就是非0为1，非非0为0。补码有一个性质，除了0和最小数的补码十七本身，其他数与其补码互为相反数关系。互为相反数的两个数位或的结果是负数，

10、howManyBits

题目解释：eturn the minimum number of bits required to represent x in two's complement

一个数用补码表示最少需要几位

解答：

如果是一个正数，则需要找到它最高的一位（假设是n）是1的，再加上符号位，结果为n+1

如果是一个负数，则需要转为正数

11、floatScale2

题目解释：（英文实在是太长了）求2乘一个浮点数的结果

解答：在CSAPP第二章中详细说明了在IEEE下的浮点数的表示方式：$V=(−1)^s×M×2^E$，其中s是符号（sign），M是尾数（significand），E是阶码（exponent，简写为exp，为浮点数编码的整数部分）。

这里主要要分为三种情况：①输入一个无穷大数或者一个NaN，那么结果就是这个输入值；②输入值为0或者无穷小，那么结果就是2倍的输入值再加上符号值；③当阶码加1等于255时，返回无穷大，否则返回阶码加1.

12、floatFloat2Int

题目解释：将浮点数转换为整数

解答：①如果原浮点值为0则返回0；②如果真实指数大于31，返回规定的溢出值0x80000000u；③先把小数部分（23位）转化为整数（和23比较），然后判断是否溢出：如果和原符号相同则直接返回，否则如果结果为负（原来为正）则溢出返回越界指定值0x80000000u，否则原来为负，结果为正，则需要返回其补码（相反数）

解答：依然是按照浮点数求值公式来计算
